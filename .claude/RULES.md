# Project Rules — Documentation Sync (MANDATORY)

## Rule

Every time you make ANY code change, you MUST update the project documentation to reflect reality.

This is not optional.

## Documentation scope

You must keep these in sync with the actual codebase:

* README.md
* any roadmap / todo / progress files
* any docs in /docs or /.claude
* any status or memory files

## Required updates after changes

After implementing ANY feature, fix, or refactor:

1. Mark completed items in todos
2. Add newly implemented features
3. Update "current state" sections
4. Add or update known limitations
5. Update next priorities if they changed

## Behavior rules

* Do NOT leave docs stale
* Do NOT create duplicate docs unless necessary
* Prefer updating existing files
* Write clearly and concretely (no vague summaries)
* Reflect actual code, not intentions

## Workflow requirement

For every task:

1. Implement code changes
2. Update documentation
3. Then report completion

A task is NOT complete until documentation is updated.
