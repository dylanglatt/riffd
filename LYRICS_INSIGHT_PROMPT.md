## Change 1: Auto-insert section header before first lyrics

**File**: `templates/decompose.html`, function `_formatLyrics()` (~line 1670)

**Problem**: Most songs from Genius don't include a `[Verse 1]` tag before the first lyrics. The current code sets `hitFirstSection = true` when it encounters a bare lyric line (line 1697), but never inserts a section header for it. Every other section gets a styled header — the opening verse just has naked lyrics at the top.

**Fix**: When `hitFirstSection` is false and we encounter a line that passes the junk filter (i.e., it's a real lyric), auto-insert a default section label BEFORE rendering that first line. Change the block at lines 1690-1698:

```javascript
    if (!hitFirstSection) {
      const lower = trimmed.toLowerCase();
      if (lower.includes('read more') || lower.includes('contributor') || lower.endsWith(' lyrics')
          || lower === 'embed' || lower === 'you might also like' || trimmed.length > 120) continue;
      // This is a real lyric line with no preceding section tag — insert a default header
      hitFirstSection = true;
      html += '<div class="lyrics-section-label">Verse</div>';
    }
```

That's it — one line added. The label says "Verse" (not "Verse 1") since we don't know the numbering. It uses the existing `.lyrics-section-label` CSS class.
